# /feature-plan-creator

## 阶段 1：需求确认（不写代码）

1. 阅读宪法（`.claude/constitution.md`）+ 相关设计文档
2. 通过 AskUserQuestion 澄清需求（2-3 轮：目标与场景 → 数据模型与集成 → 边界情况）
3. 用 1-2 段文字总结方案，**必须等待用户确认后才能进入阶段 2**

## 阶段 2：生成技术方案

输出 `Docs/{feature-name}/plan.md`：

1. **概述** — 目标、非目标
2. **影响模块** — 需要创建/修改的模块
3. **数据模型** — 关键数据结构
4. **流程设计** — 主流程 + 异常流程
5. **模块设计** — 职责、接口定义（按模块添加章节）
6. **测试计划** — TDD 周期列表（见下方粒度约束）
7. **宪法合规审计** — 逐条对照宪法检查

## 阶段 3：任务拆分

将方案拆分为**微任务列表**。每个任务必须满足：

1. **时间 ≤ 5 分钟** — 如果超过，继续拆分
2. **目标文件明确** — 每个任务指定要创建/修改的文件
3. **可验证** — 明确的完成标准（测试通过 / 编译成功 / 可观察行为）
4. **TDD 标记** — 功能代码任务标记 `[TDD]`，必须遵循 RED-GREEN-REFACTOR

```markdown
### 微任务列表

- [ ] [TDD] 实现 XXX 逻辑 -> `{file-path}` | 完成条件: XXX 测试通过
- [ ] [TDD] 实现 YYY 接口 -> `{file-path}` | 完成条件: YYY 测试通过
- [ ] 配置 ZZZ -> `{file-path}` | 完成条件: 编译成功
```

## 禁止操作

- 不要在需求不清晰时生成方案
- 不要跳过用户确认步骤
- 不要引入宪法禁止的模式
- 不要添加超出 MVP 范围的功能
- 不要生成粒度超过 5 分钟的任务
